Compass-API

Giving context to geospatial data

Meet Compass-API

Compass-API is a framework that allows to create a RESTful API service for publishing geospatial point geometries data without programming. The API returns the geometries contained on a ratio distance from the specified location coordinate. Data is returned in JSON format. Compass-API has the functionality for returning features using the GeoJSON specificationn for easy integration and rendering with multiple map libraries.

This project was developed for the TechSummitPR as part of the National Day of Civic Hacking 2013.

Requirements and Installation

Compass-API requires:

  • PostgreSQL 1.9
  • PostGIS 2.0+
  • Ruby 1.9.3

PostgreSQL with the PostGIS extensions must be installed and enabled on your database.

Database connection parameters are defined on the config/database.yml file.

Compass-API can be installed by simply downloading or cloning this repository. Once on the root directory, execute bundle install.

Technology behind Compass-API

Built with Ruby using:

  • Goliath - Non-blocking web server framework
  • Grape - Micro-framework for creating REST-like APIs
  • RGeo - Geospatial data library

How does it work?

An API is built in three single steps:

1. Import your Shapefiles

Once your PostgreSQL and PostGIS are configured and enabled, a shapefile can be imported using the shp2psql command line tool:

$ shp2psql -s 4326 myshapefile.shp table_name

Projection must be specified using the -s SRID parameter. Compass-API requires all layers to be defined using the WGS84 projection (EPSG:4326). Reprojection can be handled by shp2pgsql by specifying the current and the desired projections: -s SRID:NEW_SRID.

$ shp2psql -s 2866:4326 myshapefile.shp table_name

Encoding can be specified using the -W parameter.

Note: Compass-API only works with POINT geometries.

2. Define your API using YAML

API resources are defined on a YAML file without having to do any programming. You must edit the config/api.yml file like this:

---
- :resource: hydrant
  :table_name: hydrant
  :geometry_column: geom
  :columns:
  - :facilityid
  - :owner
  - :watertype

:resource - Defines the URL resource name.

:table_name - Specifies the database table for your resource.

:geometry_column - Specifies the geometry column (Projection must be WGS84)

:columns - List of table columns that will be included as attributes for each geometry.

3. Start Compass-API

Execute ruby server.rb -vs on the Compass-API directory.

API resource structure

Resources are defined with the following structure: /compass/[:resource]?lnglat=0.0,0.0

The lnglat or latlng parameter must be always specified.

An optional dist parameter is available for specifying the ratio distance in miles. If not specified, defaults to 1.

A feature resource is available for returning data as a GeoJSON FeatureCollection:

/compass/[:resource]/feature?lnglat=0.0,0.0
